gnuplot

1998-3-28

gnuplot-3.5(pre 3.6) beta 340 (+1.1.3)

GIFが直接出力可能なgnuplotのバージョンをインストールする。
  • ソース：gnuplot-beta340.tar.gz,gnuplot3.6b340+1.1.3.patch.tar.gz
  • ホームページ：http://www.ipc.chiba-u.ac.jp/~yamaga/gnuplot+/index-j.html 
% gzip -cd gnuplot-beta340.tar.gz | tar xvf -
% cd gnuplot
% gzip -cd gnuplot3.6b340+1.1.3.patch.tar.gz | tar xvf -
% patch -p0 < gnuplot3.6+1.1.patch
PNGライブラリが古いので、後でインストールすることにとりあえず使わないことにする
% ./configure --without-png
% make
% make -n install
# make install
で完了。
/usr/local/ 以下に
bin/gnuplot , man/man1/gnuplot.1 , share/gnuplot.gih がインストールされる。
これだけでは、'-*-fixed-medium-r-normal--14-*'というフォントが無いためエラーが出て使えない。とりあえず、~/.Xdefaults で 
Gnuplot*font:    -*-medium-r-normal--14-*
とすることで使えるようにはなる。

1997-12-8

ftp://ftp.ipc.chiba-u.ac.jp/pub/pub.yamaga/gnuplot/3.5/ から、gnufit12.tar.gz, gnuplot-3.5.tar.gz, gnuplotj.doc.gz をとってくる。 
gnufit12.tar.gzを解凍・展開して、./diff となる。
つぎに、その中で、GNUPLOT自体を解凍・展開して、パッチを当てる。
% gzip -cd gnufit12.tar.gz | tar xvf -
% cd ./diff
% gzip -cd gnuplot-3.5.tar.gz | tar xvf -
% ./install ./gnuplot
ここで、エラーが発生。
install というデフォルトのパッチスクリプト中の patch コマンドのオプションが合わなかったため、最終行を
patch -p < patch.35
から、
patch -p0 < patch.35
に変更したら、パッチ当て成功。

0INSTALL という文書の通り、
% cp makefile.unx Makefile
% make sgix11
としたところ、readline.c のコンパイルでこける。

Makefileなどをいじったり、調べたりするのがめんどくさかったので、別の手段を考えることに。
ここまでの作業を全て中止し、SGI の WWW サイトで、gnuplot で検索をかけたところ、SGI が配布しているGNUPLOT v.3.5 があることが判明。（http://www.sgi.com/TasteOfDT/public/freeware2.0/fw_gnuplot/）fw_gnuplot.tar.Z を持ってくる。

あとは、ソフトウェア・マネージャを使って、ファイルメニュー→開く…→ファイル指定→開始ボタン、の順でデフォルトのインストールが完了。/usr/freeware 以下のディレクトリに実行ファイル、マニュアルなどがおかれる。
注：前述のパッチ類はとりあえず、当てていない。また、暇があれば試してみようかしら。

Mule

1997-12-13

Mule 2.3 （Emacs19.28）

ずっと悩んでいた Mule-2.3 のインストールだったが、http://murasakinoue.esi.yamanashi.ac.jp/~oota/O2/installmemo.htmlを頼りに、インストールしてみた。
  1. ftp://etlport.etl.go.jp/pub/mule/より、mule-2.3.tar.gz, emacs-19.34b.tar.gz を入手。（ただし、 emacs-19.34b.tar.gz に関しては、ヘッダファイルを一つ利用しただけ。）
  2. % gzip -cd mule-2.3.tar.gz | tar xvf -
  3. % gzip -cd emacs-19.34b.tar.gz | tar xvf -
  4. % mv emacs-19.34/src/s/irix6-0.h mule/src/s
  5. % cd mule
  6. % ./configure --with-wnn6 \
    --with-x --with-x-toolkit \
    --with-wnn-includes=/usr/include/wnn \
    --with-wnn-libraries=/usr/lib/Wnn6
  7. % make
  8. ここで、wnnfns.c のコンパイル中にWnn のヘッダファイルが無いというエラーが出るので、強制的に
    % cc -c -Demacs -DHAVE_CONFIG_H -I. -I/usr/people/mule/src -I/usr/include/wnn -cckr -g wnnfns.c
    としてコンパイルする。
  9. もう一度、 % make
  10. root になってから、# make install
  11. これで作業完了だが、なぜか、% mule とすると、mule - コマンドが見つかりません というエラーメッセージが出た。
  12. そこで、% /usr/local/bin/mule-19.28 とすると、きちんと立ち上がった。そのあとで、% mule としても先ほどの現象は再現できず。一応、インストール完了。
